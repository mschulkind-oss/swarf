# Agent Guidelines

## Commits

Commit after every logical change — don't batch unrelated work into one commit.

Write clear, descriptive commit messages:
- Use conventional commit prefixes: `feat:`, `fix:`, `refactor:`, `test:`, `chore:`, `docs:`
- Lead with *what changed and why*, not a list of files
- Keep the first line under 72 characters
- Add a blank line and body paragraph for non-trivial changes

Do not add AI attribution trailers (Co-Authored-By, Generated by, etc.).
